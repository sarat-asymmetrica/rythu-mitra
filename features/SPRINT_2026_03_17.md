### Wave 2: Ananta Intelligence Depth (P2)

Ported the Go mathematical substrate to TypeScript, with Telugu farming adaptations.

- **`ananta.ts`** (669 LOC) — Complete Vedic reasoning layer:
  - Digital root classifier: `digitalRoot(n)` via `1 + (n-1)%9` — O(1) Vedic math
  - String DR: `digitalRootString(s)` — UTF-16 char code sum → digital root
  - Semantic cluster mapping: `{1,4,7}→action`, `{2,5,8}→analysis`, `{3,6,9}→synthesis`
  - Intent classifier: 5-way (record/query/advice/search/social) combining DR + 70+
    Telugu keyword sets + question pattern regex
  - 4 Ananta validation gates: Sanity (range ₹1→₹10L), Consistency (kind↔description
    keywords), Duplicate (5-minute window, exact paise match), Quality (harmonic mean ≥ 0.7)
  - Harmonic mean punishes single weak gates — mirrors Go's `vedic.HarmonicMean`
  - Emotional awareness: 5-state detection (worried/excited/confused/frustrated/neutral)
    with Telugu comfort phrases (`చింతించకండి`, `బాగుంది! అభినందనలు!`)
  - Pattern detection: weekly spending rhythm, month-over-month category trends,

## Critical Bug Fixed: App.svelte executeAction Bypass

### The Problem

Day 1 shipped CRUD infrastructure (action extraction, ConfirmCard, validation gates)
but the full end-to-end write path was broken by a silent architectural gap.

`App.svelte` had an inline `recordMoneyFromChat()` function that:
1. Wrote to STDB directly (correct behavior)
2. Bypassed `executeAction()` in `actions.ts` (wrong)
3. Never called `setUndo()` — undo button never appeared
4. Used `'rabi_2025'` as the season string instead of `'rabi_2026'`
5. Used a different idempotency key format

`record_crop` was already going through `executeAction()`. Only `record_money` and
`record_from_bill` had the bypass. This meant two code paths for the same operation,
diverging silently.

### The Fix

All action types now route exclusively through `executeAction()`:

```typescript
// App.svelte handleChatAction() — after fix
if (
  action.action === 'record_money' ||
  action.action === 'record_from_bill' ||
  action.action === 'record_crop' ||
  action.action === 'update_money' ||
  action.action === 'delete_money' ||
  action.action === 'remember' ||
  action.action === 'check_scheme'
) {
  const result = await executeAction(action);
  showToast(result.message, result.success ? 'default' : 'warning', 4000);
}
```

`executeAction()` provides: STDB write → `setUndo()` → Telugu toast message.
The undo countdown now appears correctly after every confirmed write.

### Data Flow (Verified)

```
ChatInput.onsend(text: string)
  -> ChatPanel.handleSend(text)
  -> chat.ts:sendMessage() -> ChatAction[] (action: string, [key]: unknown)
  -> ChatPanel: pendingAction = actions[0]
  -> ConfirmCard: action prop
  -> ConfirmCard.handleConfirm()
  -> ChatPanel.handleConfirm(action)
  -> App.svelte.handleChatAction(action)
  -> executeAction(action)                // actions.ts
  -> handleRecordMoney()                  // STDB write + setUndo() + toast
```

TypeScript structural typing ensures `chat.ts:ChatAction` and `actions.ts:ChatAction`
(both `{ action: string; [key: string]: unknown }`) are compatible without casting.

  Need a dedicated module for production with proper auth and backup.
- **PM-KISAN real API** — Currently uses Sarvam chat to simulate status. Real
  integration requires farmer's Aadhaar-linked data from PM-KISAN portal API.

## Architecture Decisions Made (Day 2)

| Decision | Choice | Reason |
|----------|--------|--------|
| OCR provider | Mistral Vision (primary), Sarvam (fallback) | No 30MB download, handles blurry photos natively |
| Offline persistence | localStorage (queue), IndexedDB (future) | localStorage sufficient for action queue; IndexedDB for full offline-first |
| PDF generation | pdf-lib on device | Zero server calls, works offline, 477 LOC clean implementation |
| Onboarding | Conversational (6-step chat) | Consistent with product thesis: chat-native interface |
| Search fallback | 4-provider chain | Resilience without requiring any API keys for basic functionality |
| Validation | Harmonic mean of 4 gates | Ported directly from Go — punishes single weak gate |
